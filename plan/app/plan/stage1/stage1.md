# Stage 1. Kanji database preparation.

## Resources
Following resources can be used to get kanji for each level, it's meanings, readings and usage examples:
1. http://www.studykanji.net/kanjilist/
2. https://jlptsensei.com/jlpt-n5-kanji-list/
3. https://jlptstudy.net/N3/
4. https://www.kanshudo.com/collections/jlpt_kanji
5. Any other suitable resourses

## Contents
The database should contain kanji for JLPT levels N5 N4 N3 N2 it's readings, translations, usage examples and level.

## Format
The database should be made in readable markdown format most suitable for this dataformat. Possible option: table.

## Checks
1. The contents of the base should be cross checked over all the data sources
2. The kanji counts should be around same as in web resources describing the JLPT exams contents
3. Same kanji should not be repeated in different levels
4. All kanji must have translation meaning and reading

## Structure
The database should be contained in the `kanji` directory in the projects's base folder. The `kanji` directory should contain N2, N3, N4, N5 subfolders should contain level.md file example: N4.md containing kanji for the according level.
